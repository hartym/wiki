## Design Patterns

Must read.

[Design Patterns: Elements of Reusable Object-Oriented Software - Amazon](http://www.amazon.fr/Design-Patterns-Elements-Reusable-Object-Oriented/dp/0201633612)

If you need to remember only one thing, it should be the two rules of Object Oriented Programming:

* Favor **object composition** over **class inheritance**.
* Program to an **interface**, not an **implementation**.
